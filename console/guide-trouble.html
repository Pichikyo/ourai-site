🎛 往来 サイト管制室困ったとき

困ったとき

直したのにサイトが変わらない

  • 同期（sync）・push（CI経由デプロイ）まで済んでいるか確認。台帳を直しただけでは反映されません
  • gh run list --workflow=deploy.yml でGitHub Actionsが成功しているか確認できます
  • ブラウザのキャッシュが古いことも。強制再読み込み（Cmd+Shift+R）を一度

間違えて壊したかも

  • 往来サイト側（~/dev/ourai-site/）はGitで履歴管理されているので、git log / git diff でどの時点にも戻せます。慌てて上書きせず、その状態のまま相談を
  • 台帳・同期スクリプト側（~/inhouse-bot/、台帳の正本が置いてある場所）もGitで管理されているので、同様に cd ~/inhouse-bot && git log で履歴を確認し、必要なら該当ファイルだけ過去のコミットから復元できます
  • どちらのリポジトリを戻すべきかで対処が変わるため、まず「サイト側の見た目が壊れた」のか「台帳（inhouse-bot側）の中身が壊れた」のかを切り分けてから復旧してください

往来には引き渡し予定のバックアップZIP（MiTERU案件のような *-site.zip / *-scheme-files.zip）を作っていません。復旧手段は上記のGit履歴のみです。

判断に迷う

  • 載せていいか迷う内容・文言・カテゴリの判断は、社内台帳の正本（~/inhouse-bot/ourai-works-ledger.json / ourai-news-ledger.json）を確認するか、智美さんに相談してください
